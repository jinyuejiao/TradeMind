# 产品中心界面优化计划

## 问题分析

根据用户需求，需要对产品中心界面进行以下优化：

1. **界面精简与布局全屏化**：
   - 移除冗余模块：AI 选品预测池、库存健康报表、产研关键进度
   - 重构布局：将 grid-cols-3 改为单列 w-full
   - 按钮替换：将右上角按钮改为“生成进货单据”

2. **实现“进货建议”逻辑与弹窗**：
   - 实现 openPurchaseSuggestionModal 函数
   - 构建弹窗 UI，按供应商分组显示进货建议

3. **手机端响应式深度适配**：
   - 表格转卡片
   - 弹窗适配
   - 底部操作栏

## 修复方案

### 1. 界面精简与布局全屏化
- 移除冗余模块：删除 AI 选品预测池、库存健康报表、产研关键进度
- 重构布局：将原本的 grid-cols-3 结构改为单列 w-full
- 按钮替换：将右上角的按钮修改为“生成进货单据”按钮

### 2. 实现“进货建议”逻辑与弹窗
- 实现 openPurchaseSuggestionModal 函数：
  - 调用 wrappedFetch('/api/v1/rd/products') 获取全量产品
  - 筛选出 stock <= warning_stock 的产品
  - 按 supplier_id 分组
  - 计算建议进货量：(warning_stock * 2) - stock
- 构建弹窗 UI：
  - 使用 id="purchase-suggestion-modal"
  - 按供应商显示卡片，每个卡片内部是一个产品明细表
  - 展示字段：供应商名称、产品名 (SKU)、缺货状态、建议采购、预估小计

### 3. 手机端响应式深度适配
- 表格转卡片：在手机端 (sm:hidden)，将产品库表格转换为卡片流展示
- 弹窗适配：
  - 增加 max-sm:h-full max-sm:rounded-none 类名，实现手机端全屏覆盖
  - 将弹窗内的横向表格在手机端改为“缩略图 + 标题 + 输入框”的垂直布局
- 底部操作栏：在手机端，将“保存进货单”按钮固定在弹窗底部，背景使用 backdrop-blur

## 修复步骤

1. **界面精简与布局全屏化**
   - 移除冗余模块
   - 重构布局
   - 按钮替换

2. **实现“进货建议”逻辑与弹窗**
   - 实现 openPurchaseSuggestionModal 函数
   - 构建弹窗 UI

3. **手机端响应式深度适配**
   - 表格转卡片
   - 弹窗适配
   - 底部操作栏

4. **测试修复效果**
   - 测试界面布局
   - 测试进货建议功能
   - 测试手机端适配

## 预期结果

- 产品中心界面更加简洁，产品库扩展至全屏
- 点击“生成进货单据”按钮能够显示按供应商分组的进货建议
- 界面在手机端能够正常显示和操作

## 风险评估

- **低风险**：修改主要是界面优化和功能添加，不会影响现有核心功能
- **中风险**：需要确保 API 调用和数据处理逻辑正确
- **低风险**：所有修改都在现有的代码结构基础上进行，不会破坏现有功能
