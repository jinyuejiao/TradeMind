# 产品中心UI完善实施计划

## 项目背景分析

通过对现有代码的分析，当前产品中心页面已具备基本的产品展示功能，但需要进一步完善以下功能：

1. **现有产品库**：默认展示全量产品信息，按销量排序
2. **筛选功能**：增加产品类别、供应商、库存情况筛选的UI交互代码
3. **手机端适配**：将适配代码抽象至 `./MobileAdapt` 目录

## 实施计划

### Step 1: 创建产品数据模型和初始化逻辑

**修改/创建文件**：
- 创建或修改产品数据模型文件，包含销量字段
- 添加产品初始化逻辑，按销量排序展示产品

**具体任务**：
1. 定义产品数据结构，包含销量字段
2. 准备示例产品数据，包含销量信息
3. 实现按销量排序的初始化逻辑

### Step 2: 实现筛选器UI交互代码

**修改文件**：
- `modules/product-center/product-center.html`

**具体任务**：
1. 为三个筛选按钮添加下拉菜单UI
2. 产品类别筛选：
   - 二级级联菜单（一级分类 -> 二级分类）
   - 使用现有的分类数据
3. 供应商筛选：
   - 供应商列表下拉菜单
4. 库存情况筛选：
   - 选项：全部/充足/预警/缺货
5. 实现筛选器的展开/收起交互
6. 实现筛选器选中状态的UI反馈

### Step 3: 实现筛选逻辑

**创建/修改文件**：
- 创建 `JsImpl/product-center.js` 文件（如不存在）或修改现有JS文件
- `modules/product-center/product-center.html`

**具体任务**：
1. 实现产品筛选函数，支持：
   - 按产品类别筛选（支持二级分类）
   - 按供应商筛选
   - 按库存情况筛选
2. 实现筛选结果的实时更新
3. 支持多条件组合筛选

### Step 4: 完善手机端适配，抽象代码至MobileAdapt

**修改文件**：
- `MobileAdapt/mobile.css`
- `modules/product-center/product-center.html`

**具体任务**：
1. 在 `mobile.css` 中添加产品中心专用的移动端样式
2. 确保筛选器在手机端的良好显示和交互
3. 确保产品表格在手机端的适配
4. 确保产品卡片在手机端的适配
5. 确保所有弹窗在手机端的适配

### Step 5: 测试和验证

**具体任务**：
1. 测试产品列表按销量排序功能
2. 测试各个筛选器的功能
3. 测试多条件组合筛选
4. 测试手机端适配效果
5. 测试响应式布局

## 技术实现要点

1. **数据模型**：使用JavaScript对象数组存储产品数据，包含销量字段
2. **筛选逻辑**：使用数组的 `filter` 和 `sort` 方法实现筛选和排序
3. **UI更新**：使用DOM操作动态更新产品列表
4. **移动端适配**：使用Tailwind CSS的响应式类和自定义CSS
5. **事件处理**：使用事件监听器处理筛选器交互

## 预期成果

完成后，产品中心将具备：
1. 按销量排序的产品列表
2. 功能完整的筛选系统（产品类别、供应商、库存情况）
3. 良好的手机端适配效果
4. 抽象的适配代码结构，便于维护
