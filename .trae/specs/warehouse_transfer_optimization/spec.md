# 仓库调拨弹窗优化 - Product Requirement Document

## Overview
- **Summary**: 对现有的仓库调拨弹窗界面进行两项关键优化：1）将调拨类型选择从药丸组改为仅变价调拨的圆形勾选框；2）为产品列表添加新增行功能，支持通过下拉框选择源仓库的产品。
- **Purpose**: 简化调拨类型选择交互，提供更灵活的产品选择方式，提升用户体验。
- **Target Users**: 商贸企业仓库管理人员、产品中心操作人员

## Goals
- 将调拨类型选择简化为仅变价调拨的圆形勾选框（默认不勾选=同价调拨）
- 为产品列表添加新增行功能
- 新增行的产品列使用下拉框，可选择源仓库的产品
- 保持与现有系统风格一致
- 确保移动端适配良好

## Non-Goals (Out of Scope)
- 不修改核心调拨逻辑（价格计算、确认提交等）
- 不实现真正的库存扣减
- 不修改确认调拨后的数据流

## Background & Context
- 现有仓库调拨弹窗已实现基础功能
- 当前使用药丸组（Pills）样式选择同价/变价调拨
- 产品列表目前仅显示硬编码的产品
- 已有完整的源仓库产品数据（window.TM_MOCK_WAREHOUSE_STOCKS）
- 系统使用 Tailwind CSS 和 Phosphor Icons

## Functional Requirements
- **FR-1**: 移除药丸组样式的调拨类型选择
- **FR-2**: 添加仅变价调拨的圆形勾选框，不勾选默认为同价调拨
- **FR-3**: 在产品列表头部添加"新增行"按钮
- **FR-4**: 点击新增行可以添加新的产品行
- **FR-5**: 新增行的产品列使用下拉框，显示源仓库的所有产品
- **FR-6**: 选择产品后自动填充默认单价和数量
- **FR-7**: 支持删除新增的产品行

## Non-Functional Requirements
- **NFR-1**: 勾选框使用圆形样式，符合系统设计风格
- **NFR-2**: 新增行按钮使用 teal-500 配色
- **NFR-3**: 产品下拉框样式与目标仓库下拉框一致
- **NFR-4**: 所有操作有适当的 hover 效果和过渡动画
- **NFR-5**: 移动端适配良好

## Constraints
- **Technical**: 必须使用现有的技术栈（HTML5, Tailwind CSS, Vanilla JavaScript）
- **Business**: 必须与现有产品中心的设计风格完全一致
- **Dependencies**: 依赖现有的 window.TM_MOCK_WAREHOUSE_STOCKS 数据

## Assumptions
- 源仓库产品数据已完整（window.TM_MOCK_WAREHOUSE_STOCKS）
- 新增行后可以动态计算总价值
- 用户可以删除自己新增的产品行

## Acceptance Criteria

### AC-1: 调拨类型选择简化为勾选框
- **Given**: 用户打开仓库调拨弹窗
- **When**: 查看调拨类型选择区域
- **Then**: 
  - 不再显示药丸组样式
  - 显示一个圆形勾选框，标签为"变价调拨"
  - 默认状态为未勾选（表示同价调拨）
  - 勾选后单价输入框可以编辑
- **Verification**: `human-judgment`

### AC-2: 产品列表新增行按钮
- **Given**: 用户打开仓库调拨弹窗
- **When**: 查看产品清单区域
- **Then**: 
  - 在产品清单标题右侧有一个"新增行"按钮
  - 按钮样式符合系统设计风格
  - 按钮有适当的 hover 效果
- **Verification**: `human-judgment`

### AC-3: 点击新增行添加新产品行
- **Given**: 用户在仓库调拨弹窗中
- **When**: 点击"新增行"按钮
- **Then**: 
  - 在产品列表底部添加一个新的空行
  - 新行的产品列是一个下拉框
  - 下拉框显示源仓库的所有可用产品
  - 新增行可以正常编辑
- **Verification**: `human-judgment`

### AC-4: 产品下拉框选择产品
- **Given**: 用户点击了新增行
- **When**: 在新增行的产品下拉框中选择一个产品
- **Then**: 
  - 产品名称列显示选中的产品名称和 SKU
  - 单价列自动填充该产品的默认单价
  - 数量列显示默认数量（如 1）
  - 总价列自动计算并显示
  - 总调拨价值自动更新
- **Verification**: `human-judgment`

### AC-5: 支持删除新增的产品行
- **Given**: 用户已新增了至少一行产品
- **When**: 点击新增行的删除按钮
- **Then**: 
  - 该新增行被删除
  - 剩余行的顺序保持不变
  - 总调拨价值自动更新
- **Verification**: `human-judgment`

### AC-6: 移动端适配良好
- **Given**: 用户在移动设备上访问（屏幕宽度 < 768px）
- **When**: 查看和操作调拨弹窗
- **Then**: 
  - 勾选框在移动端显示正常
  - 新增行按钮大小适中，易于点击
  - 产品下拉框在移动端易于操作
  - 所有元素布局合理，不拥挤
- **Verification**: `human-judgment`
